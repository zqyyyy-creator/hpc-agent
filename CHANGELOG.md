# Changelog

## 0.2.5 - 2026-07-13

### Added
- Added a general MCP server entry point, `hpc-agent-mcp`, with STDIO and Streamable HTTP transports.
- Added MCP natural-language entry tool `hpc_agent_chat`, sharing the TUI intent routing and conversation context.
- Added MCP Slurm tools for config checks, cluster knowledge lookup, sbatch preview, guarded submit, job query, cleanup preview, and guarded cleanup execution.
- Added MCP VASP tools for job preview, input generation, local result analysis, output sync, and structured VASP workflows.
- Added structured MCP tools for generic clients: `hpc_generate_sbatch_structured`, `hpc_prepare_vasp_job_structured`, `vasp_generate_inputs_structured`, `vasp_analyze_local_result_structured`, `vasp_sync_output_structured`, `hpc_query_job_structured`, and `hpc_prepare_cleanup_structured`.
- Added MCP resources for capabilities, tool schema, examples, security policy, deployment status, recent jobs, skills, config status, user guide, and cluster information.
- Added MCP prompt templates for safe Slurm submit, Slurm diagnosis, VASP workflow, local VASP analysis, cleanup, natural-language use, and connection debugging.
- Added Streamable HTTP `/health` endpoint, audit logging, host/origin allow-list support, and guarded execution environment switches.
- Added `scripts/start_mcp_http.sh` and a user-level `systemd` service template for repeatable MCP HTTP startup.
- Added dedicated MCP documentation under `docs/mcp_docs/`, including generic MCP client guidance, ChatGPT Web setup, app metadata, deployment, and safety notes.
- Added anonymous public-download installation documentation for hpc-agent 0.2.5.
- Added Docker documentation updates for version 0.2.5.

### Changed
- Updated package version to `0.2.5`.
- Updated packaging metadata to include MCP dependencies, console entry point, MCP docs, startup script, and systemd template.
- Updated release layout and installation page to reflect免密匿名下载 without repository username/token.
- Improved Slurm request parsing for labeled structured text such as `command:`, `nodes:`, `time:`, `partition:`, CPU, memory, and GPU fields.
- Improved routing so explicit Slurm script preview requests are not misclassified as parameter suggestions or test-file generation.
- Improved VASP local analysis path compatibility for job directories, `raw_output` directories, and output collection directories.

### Fixed
- Fixed MCP/ChatGPT Slurm preview cases where `command: hostname` was not recognized as the execution command.
- Fixed Streamable HTTP tunnel connection issues by documenting and supporting explicit `--allowed-host` configuration.
- Fixed startup-script `.env` loading so values containing spaces, such as shell `source ...` setup commands, do not break MCP service startup.
- Fixed VASP analysis feedback to distinguish an output collection directory from a single analyzable VASP job directory.

### Compatibility
- Required Python: 3.12+
- Main TUI command: `hpc-agent`
- Config check command: `hpc-agent-check`
- Initializer command: `hpc-agent-init`
- MCP command: `hpc-agent-mcp`

### Notes
- MCP is a general external-client interface, not a ChatGPT-only integration.
- Guarded MCP actions remain disabled by default. Real submit/sync requires `confirm=true` and `HPC_AGENT_MCP_ENABLE_WRITE=1`; destructive cleanup requires `confirm=true` and `HPC_AGENT_MCP_ENABLE_DESTRUCTIVE=1`.
- VASP `POTCAR` files are still never generated by HPC Agent. Users must provide authorized `POTCAR` files themselves.

## 0.2.0 - 2026-06-18

### Added
- Added Textual TUI as the unified interactive entry point.
- Added ordinary Slurm job generation, preview, confirmed submit, query, monitor, diagnosis, and cleanup workflows.
- Added local job registry support for recent jobs, job details, VASP job listing, archive preview, archive confirmation, and archive restore.
- Added VASP fixed-directory submit workflow with local input, remote input, remote output, output sync, and one-shot analysis.
- Added VASP report generation through Claude Code with `report.md`, `paper_methods.md`, and `paper_results.md`.
- Added deterministic VASP output parsing and `report_context.md` generation from synced outputs.
- Added VASP input generation from an existing authorized `POTCAR`, including `INCAR`, `KPOINTS`, and `POSCAR`.
- Added VASP input parameter overrides for calculation type, `ENCUT`, `KPOINTS`, `NSW`, `EDIFF`, `ISMEAR`, and `SIGMA`.
- Added overwrite confirmation for existing VASP input files.
- Added repeated VASP run handling with overwrite old results, automatic new run name, or cancel options.
- Added config diagnostics and recovery suggestions for `.env`, SSH keys, local and remote directories, VASP commands, partitions, and Claude Code/API settings.
- Added unified error diagnosis with real error cases first and generic error patterns as fallback.
- Added semi-automatic real error case drafting and confirmed writes to `data/errors/real_cases.json`.
- Added `/help`, `/help job`, and `/help vasp` shortcut help entries.
- Added selectable TUI chat text and `Ctrl+Y` copy behavior for selected text or the latest Agent reply.

### Changed
- Updated user documentation and docs entry points for the current Slurm, VASP, TUI, config, and error-diagnosis workflows.
- Reworked the error knowledge base into `real_cases.json` and `generic_errors.json`.
- Hardened VASP overwrite behavior so old remote input/output and local output directories are cleared before resubmission when the user chooses overwrite.
- Improved local job lifecycle behavior, including recent job ordering and job detail presentation.
- Expanded `.gitignore` coverage for local secrets, generated job records, VASP licensed inputs, and large VASP output artifacts.

### Fixed
- Fixed VASP submit selection so an explicitly requested job name is not replaced by the most recent complete VASP directory.
- Fixed "analyze the previous job" handling so the Agent can resolve the latest relevant job record.
- Fixed recent job listing so Slurm test jobs are not hidden behind newer VASP output directory mtimes.
- Fixed VASP input overwrite confirmation so replies such as `确认覆盖` and `覆盖已有配置文件` execute the pending overwrite instead of asking again.
- Fixed remote VASP cleanup behavior and documentation so input/output cleanup semantics are explicit.

### Notes
- HPC Agent does not generate real VASP `POTCAR` files. Users must provide `POTCAR` files from a VASP pseudopotential source they are authorized to use.
- This release is intended as a source-based milestone release. Clone the repository, configure `.env`, run `uv sync`, and start with `.venv/bin/python app.py`.
