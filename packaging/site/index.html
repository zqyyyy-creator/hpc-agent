HPC Agent 安装与更新说明

本页面用于指导用户从火山私服下载、安装、更新和检查 HPC Agent。安装前只需要本机已有 Python 3.12 或更高版本。

当前版本: 0.2.0 Python: 3.12+ 私服: Volcengine Artifacts

一、安装前检查

打开终端，先检查电脑里是否已经安装 Python。

python3 --version

如果看到类似下面的输出，说明 Python 可用：

Python 3.12.0
如果版本低于 3.12，或者提示 command not found，需要先安装 Python 3.12 或更高版本。

二、准备私服认证信息

私服下载需要用户名和 token。用户名可以写入终端，token 不要写到网页、文档或截图里。

read -p "请输入私服用户名: " VOLC_USERNAME
read -s VOLC_TOKEN

先输入自己的私服用户名并按回车。执行 read -s VOLC_TOKEN 后，粘贴自己的 token 并按回车。终端不会显示 token，这是正常现象。

请不要把用户名或 token 明文写进 HTML、脚本、聊天记录或公开文档。这个页面不会保存任何账号信息。

三、下载安装文件

先下载安装脚本，再下载 wheel 安装包。由于私服需要认证，推荐先下载到本地，再执行安装。

1. 下载安装脚本

curl -fL \
  -u "$VOLC_USERNAME:$VOLC_TOKEN" \
  "https://artifacts-cn-beijing.volces.com/repository/agents/models/ai-llm/hpc-agent/stable/install.sh" \
  -o /tmp/hpc-agent-install.sh

2. 下载 wheel 安装包

curl -fL \
  -u "$VOLC_USERNAME:$VOLC_TOKEN" \
  "https://artifacts-cn-beijing.volces.com/repository/agents/models/ai-llm/hpc-agent/stable/hpc_agent-0.2.0-py3-none-any.whl" \
  -o /tmp/hpc_agent-0.2.0-py3-none-any.whl

四、安装 HPC Agent

执行下面命令安装。安装脚本会创建独立虚拟环境、安装 HPC Agent、创建命令入口、初始化配置文件，并自动运行体检。

HPC_AGENT_WHEEL=/tmp/hpc_agent-0.2.0-py3-none-any.whl \
sh /tmp/hpc-agent-install.sh

如果最后看到下面这行，就说明安装流程完成：

HPC Agent installation complete.

五、运行安装检查

如果终端提示 hpc-agent-check 找不到，先把用户命令目录加入当前终端的 PATH。

export PATH="$HOME/.local/bin:$PATH"
hpc-agent-check

体检结果里重点确认这些项目是 OK：

  • data/hpc_documents 存在
  • skills 存在
  • docs 存在
  • hpc-agent entrypoint 可 import、可调用
  • RAG source files 正常
  • Skill handlers 全部可 import
第一次安装后出现 .env、SSH key、HPC 路径相关 WARN 是正常的，说明还没有填写真实配置，不代表安装失败。

六、填写配置

首次安装会创建配置文件：

~/.config/hpc-agent/.env

编辑配置文件：

nano ~/.config/hpc-agent/.env

至少需要填写这些配置：

PARATERA_BASE_URL模型 API 地址。
PARATERA_API_KEY模型 API Key。
HPC_HOST超算登录地址。
HPC_USERNAME超算用户名。
HPC_KEY_PATHSSH 私钥路径。
HPC_REMOTE_WORKDIR超算上的工作目录。
HPC_VASP_REPORT_MODELVASP 报告生成使用的模型，默认 DeepSeek-V4-Pro。

如果需要使用 VASP 功能

如果用户要提交、同步或分析 VASP 作业，还需要配置本地 VASP 输入目录和输出目录。这两个目录需要在本机存在，并且当前用户可读写。

如果用户要使用“分析 VASP 作业”或“生成 VASP 报告”功能，还需要确认 HPC_VASP_REPORT_MODEL 已填写为网关支持的模型名。

HPC_LOCAL_VASP_JOBS_INPUT_DIR本地 VASP 输入文件目录，用于放置待提交的 VASP 作业输入。
HPC_LOCAL_VASP_JOBS_OUTPUT_DIR本地 VASP 输出文件目录，用于保存从超算同步回来的 VASP 结果。
HPC_VASP_REPORT_MODELVASP 作业分析和报告生成使用的模型，例如 DeepSeek-V4-Pro。
mkdir -p ~/vasp-jobs-input ~/vasp-jobs-output

然后在 ~/.config/hpc-agent/.env 中填写：

HPC_LOCAL_VASP_JOBS_INPUT_DIR=/home/你的用户名/vasp-jobs-input
HPC_LOCAL_VASP_JOBS_OUTPUT_DIR=/home/你的用户名/vasp-jobs-output
HPC_VASP_REPORT_MODEL=DeepSeek-V4-Pro

如果使用 nano 编辑器，保存方式是：按 Ctrl + O，回车确认，再按 Ctrl + X 退出。

七、启动 HPC Agent

配置完成后，运行：

hpc-agent

如果命令找不到，先执行：

export PATH="$HOME/.local/bin:$PATH"

八、更新 HPC Agent

更新方式和安装类似：重新下载 stable 安装脚本和 wheel，然后执行安装。配置文件默认不会被覆盖。

curl -fL \
  -u "$VOLC_USERNAME:$VOLC_TOKEN" \
  "https://artifacts-cn-beijing.volces.com/repository/agents/models/ai-llm/hpc-agent/stable/install.sh" \
  -o /tmp/hpc-agent-install.sh

curl -fL \
  -u "$VOLC_USERNAME:$VOLC_TOKEN" \
  "https://artifacts-cn-beijing.volces.com/repository/agents/models/ai-llm/hpc-agent/stable/hpc_agent-0.2.0-py3-none-any.whl" \
  -o /tmp/hpc_agent-0.2.0-py3-none-any.whl

HPC_AGENT_WHEEL=/tmp/hpc_agent-0.2.0-py3-none-any.whl \
HPC_AGENT_FORCE_REINSTALL=1 \
sh /tmp/hpc-agent-install.sh

九、常见问题

curl 返回 401 或 403

通常是用户名或 token 不正确，重新执行认证步骤并确认 token 没有复制多余空格。

curl 提示 URL rejected 或 No host part

通常是命令里的 URL 没复制完整，或变量没有设置。建议直接复制本页面里的完整命令。

hpc-agent-check 显示 WARN

如果 WARN 是 .env、SSH key、HPC 路径、远端目录相关，通常只是配置还没填好。安装包本身仍然可以是正常的。

安装时提示缺少 Python 依赖

说明当前机器不能访问 Python 依赖源。请联系管理员确认网络或私服 Python 依赖源配置。

hpc-agent 命令找不到

执行下面命令后重试：

export PATH="$HOME/.local/bin:$PATH"

发布文件路径：Volcengine Artifacts / models / ai-llm / hpc-agent / stable

本页面不包含 token。请从安全渠道获取 token，并只在自己的终端中输入。
